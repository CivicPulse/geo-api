## Integration Findings

### Finding 1 — ArgoCD targetRevision (Medium)

Both ArgoCD Application CRs point to `phase-23-deploy-fix` instead of `main`. The CD pipeline pushes manifest commits to `main`, so auto-sync is broken in git state. Already tracked as TODO (commit `9d48b68`).

**Fix:** Change `targetRevision` to `main` in both `k8s/overlays/dev/argocd-app.yaml` and `k8s/overlays/prod/argocd-app.yaml`, commit, and re-apply CRs.

### Finding 2 — MIN_READY env vars not in ConfigMap (Low)

`health.py` reads `MIN_READY_GEOCODING_PROVIDERS` and `MIN_READY_VALIDATION_PROVIDERS` env vars (default 1). These are absent from the ConfigMap, making the configurable threshold invisible to operators.

### Finding 3 — Dual OTEL endpoint env vars (Low)

ConfigMap contains both `OTEL_EXPORTER_ENDPOINT` (used by app) and `OTEL_EXPORTER_OTLP_ENDPOINT` (used by SDK). Both set identically. Maintenance ambiguity if they ever diverge.
