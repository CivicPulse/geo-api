### Integration Tech Debt (Non-Blocking)

**Tiger extension check predicate — TIGR-04 edge case**

- **File:** `src/civpulse_geo/providers/tiger.py` (CHECK_EXTENSION_SQL, line ~98)
- **Issue:** `_tiger_extension_available()` queries `pg_available_extensions` (extensions available on the PostgreSQL installation) rather than `pg_extension` (extensions installed in the current database). If `postgis_tiger_geocoder` is present in the OS-level PostgreSQL installation but has never been activated with `CREATE EXTENSION`, the guard returns `True`, the provider is registered, and every Tiger geocode/validate request raises `ProviderError` at runtime.
- **Affected requirements:** TIGR-04 (edge case), TIGR-01, TIGR-02 (runtime failures in that environment)
- **Severity:** Non-blocking. The common cases (extension absent entirely → False; extension installed correctly → True) work correctly. The latent case requires a specific environment state. Caught by unit tests only via mock; live verification would surface this.
- **Fix:** Change `SELECT ... FROM pg_available_extensions WHERE name = 'postgis_tiger_geocoder'` to query `pg_extension` (installed extensions in the connected database).
