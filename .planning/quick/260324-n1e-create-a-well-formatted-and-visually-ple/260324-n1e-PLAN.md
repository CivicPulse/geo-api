| Component | Technology |

- Async driver: asyncpg
- HTTP client: httpx
- Migrations: Alembic
- CLI: Typer + Rich
- Package manager: uv
- Dev environment: Docker Compose

**Quick Start section:**
1. Prerequisites: Docker, Docker Compose
2. Clone the repo
3. `cp .env.example .env`
4. `docker compose up` — starts PostGIS + API, runs migrations and seed automatically
5. Visit http://localhost:8042/health to verify
6. Visit http://localhost:8042/docs for interactive Swagger UI

**API Endpoints section** as a table:
| Method | Path | Description |
- POST /geocode — Geocode a freeform address (cache-first)
- POST /geocode/batch — Batch geocode multiple addresses
- PUT /geocode/{hash}/official — Set official geocoding result (admin)
- POST /geocode/{hash}/refresh — Force re-query all providers
- GET /geocode/{hash}/providers/{name} — Get provider-specific result
- POST /validate — Validate and USPS-standardize an address
- POST /validate/batch — Batch validate multiple addresses
- GET /health — Service health check

Include a short usage example using curl for single geocode:
```bash
curl -X POST http://localhost:8042/geocode \
  -H "Content-Type: application/json" \
  -d '{"address": "123 Main St, Macon, GA 31201"}'
```

**CLI Commands section:**
- `geo-import import <file>` — Import GIS data (GeoJSON/KML/SHP)
- `geo-import load-oa <file.geojson.gz>` — Load OpenAddresses data
- `geo-import load-nad <file.zip> --state GA` — Load NAD data by state
- `geo-import setup-tiger <state>` — Install Tiger geocoder extensions and data (run inside db container)

Show Docker exec example for CLI:
```bash
docker compose exec api geo-import load-oa data/us_ga_bibb.geojson.gz
docker compose exec db geo-import setup-tiger GA
```

**Data Providers section:**
Brief description of each provider with a sub-table or list:

- National Address Database / NAD (local, requires data import)
- PostGIS Tiger/LINE (local, requires extension setup + state data download)
- scourgify (offline address validation/normalization)

Note: Local providers bypass the DB cache and return results directly.

**Development section:**
- Running locally with Docker: `docker compose up`
- API auto-reloads on source changes (volume mount)
- Running tests: `docker compose exec api pytest` or locally with `uv run pytest`
- Debug support: Set `DEBUG=1` in docker-compose.yml to enable debugpy on port 5680
- VS Code attach config available in `.vscode/launch.json`
