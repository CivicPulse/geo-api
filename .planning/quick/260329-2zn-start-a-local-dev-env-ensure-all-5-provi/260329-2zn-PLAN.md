<task type="auto">
  <name>Task 1: Mount tiger-data volume, create bootstrap script, start stack</name>
  <files>docker-compose.yml, scripts/dev-bootstrap.sh</files>
  <action>
1. Modify docker-compose.yml to add a bind mount of the local tiger-data/ directory into the api container at the path PostGIS loader expects. The setup-tiger command uses /gisdata as staging_fold, and the loader expects files at /gisdata/www2.census.gov/geo/tiger/TIGER2024/{TABLE}/. Add this volume to the api service:
   - ./tiger-data:/gisdata/www2.census.gov/geo/tiger/TIGER2024:ro

2. Also ensure the db container has the DEBUG env var set to "0" (it should not get debugpy). The api container already has DEBUG=1 which triggers debugpy wait-for-client. Change DEBUG to "0" in docker-compose.yml for api service so the API starts without waiting for a debugger.

3. Create scripts/dev-bootstrap.sh that:
   - Runs docker compose up -d --build
   - Waits for the api container to be healthy (poll docker compose ps or curl health endpoint, max 60s)
   - Loads OpenAddresses data: docker compose exec api geo-import load-oa /app/data/US_GA_Bibb_Addresses_2026-03-20.geojson.gz
   - Loads Tiger data: docker compose exec api geo-import setup-tiger GA (since tiger-data is mounted at the expected path, this should use the local files instead of downloading)
   - Loads NAD data: docker compose exec api geo-import load-nad /app/data/NAD_r21_TXT.zip --state GA
   - Loads Macon-Bibb data: docker compose exec api geo-import load-macon-bibb /app/data/Address_Points.geojson
   - After all loads, restarts the api container so lifespan re-checks data availability: docker compose restart api
   - Waits for api to be healthy again
   - Runs a final health check: curl http://localhost:8042/health
   - Make the script executable (chmod +x)

4. Start the stack: run docker compose up -d --build and wait for healthy status.
