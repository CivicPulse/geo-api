## Notable Architectural Decisions

**Phase 24 refactor (docker-compose.yml):** The planned shared `osm-postgres` service was dropped during manual Docker verification. Both `mediagis/nominatim:5.2` and `overv/openstreetmap-tile-server:2.3.0` bundle their own internal PostgreSQL and cannot be pointed at an external PG without custom image builds. Each sidecar now owns its PG. Isolation from `civpulse_geo` is preserved via separate volumes per service. CONTEXT.md Decision D-02 was superseded — documented in `.planning/phases/24-osm-data-pipeline-docker-compose-sidecars/24-05-SUMMARY.md`.

**Phase 28 selector collision (fixed in aedcd3b):** `commonLabels` in `kustomization.yaml` was overriding each sidecar's `selector.matchLabels`, which would have caused all 4 Deployments to fight for the same pods in a live cluster. Fix: removed `app.kubernetes.io/name` from commonLabels so each resource manifest controls its own selector.
