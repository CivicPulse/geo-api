# Phase 22: Observability — Context & Decisions

## Prior Decisions Carried Forward

- **Logging framework**: Loguru (already used in 7+ modules, no migration)
- **OTel + Loguru incompatibility**: `opentelemetry-instrumentation-logging` has no effect on Loguru — must use `logger.configure(patcher=add_otel_context)` to inject `trace_id`/`span_id` (STATE.md)
- **Observability stack targets**: Logs → stdout → Grafana Alloy → Loki; Traces → OTLP → Tempo; Metrics → `/metrics` → VictoriaMetrics (PROJECT.md)
- **Existing config**: K8s ConfigMap has `LOG_LEVEL`, `ENVIRONMENT`; `GIT_COMMIT` env var from Dockerfile build arg

## Decisions Made

### D-01: Env-based log format switching

- Add `log_format: str = "auto"` to Settings (`auto|json|text`)
- When `auto`: JSON if `ENVIRONMENT != "local"`, human-readable otherwise
- JSON output includes: `timestamp`, `level`, `message`, `request_id`, `trace_id`, `span_id`, `service`, `environment`, `module`, `function`, `line`
- Human-readable output uses Loguru's default colorized format (for local dev)
- No new ConfigMap entry needed — existing `ENVIRONMENT` drives the switch
- Implemented in a new `logging.py` module, called from `main.py` lifespan

### D-02: Full three-tier Prometheus metrics

**Tier 1 — Standard HTTP:**
- `http_requests_total` (counter: method, path, status_code)
- `http_request_duration_seconds` (histogram: method, path)
- `http_requests_in_progress` (gauge)

**Tier 2 — Service-specific:**
- `geo_provider_requests_total` (counter: provider, status)
- `geo_provider_duration_seconds` (histogram: provider)
- `geo_cascade_stages_used` (histogram) — how many stages before a result
- `geo_cache_hits_total` / `geo_cache_misses_total` (counters)

**Tier 3 — Business/LLM:**
- `geo_llm_corrections_total` (counter: model)
- `geo_llm_duration_seconds` (histogram)
- `geo_batch_size` (histogram) — batch endpoint request sizes

Library: `prometheus-client` with a `/metrics` endpoint route.

### D-03: Manual spans for cascade stages + providers

- Auto-instrumentation for FastAPI, SQLAlchemy (asyncpg), httpx via `opentelemetry-instrument-*` packages
- Manual spans at each cascade stage boundary in `CascadeOrchestrator.run()` (7 stages: normalize, spell-correct, exact-match, fuzzy, llm-correction, consensus, auto-set)
- Manual spans wrapping each `provider.geocode()` call with `provider` attribute
- Tracer obtained via `opentelemetry.trace.get_tracer("civpulse-geo")`
- OTLP exporter configured via standard `OTEL_EXPORTER_OTLP_ENDPOINT` env var (added to ConfigMap)

### D-04: Accept-or-generate request IDs

- Middleware checks `X-Request-ID` header from upstream callers
- If present, use it; if absent, generate UUID4
- Bind `request_id` to Loguru context via `logger.contextualize(request_id=...)`
- Set `X-Request-ID` response header for downstream correlation
- Health/readiness endpoints excluded from request-ID middleware
