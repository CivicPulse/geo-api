  <action>
    Create `k8s/base/deployment.yaml` with the complete pod spec. This is the most complex manifest.

    **Deployment structure:**
    - `replicas: 1` (single replica per Out of Scope)
    - `terminationGracePeriodSeconds: 30` (D-11: 10s preStop + 20s drain)
    - 5 initContainers (4 regular + 1 sidecar, sidecar MUST be last per Pitfall 1)
    - 1 main container (geo-api)
    - 1 volume (ollama-data PVC)

    **Full deployment.yaml content:**
    ```yaml
    apiVersion: apps/v1
    kind: Deployment
    metadata:
      name: geo-api
    spec:
      replicas: 1
      selector:
        matchLabels:
          app.kubernetes.io/name: geo-api
      template:
        metadata:
          labels:
            app.kubernetes.io/name: geo-api
        spec:
          terminationGracePeriodSeconds: 30
          initContainers:
            # 1. Wait for PostgreSQL to be reachable
            - name: db-wait
              image: ghcr.io/civicpulse/geo-api:latest
              command: ["sh", "-c"]
              args:
                - |
                  until pg_isready -h postgresql.civpulse-infra.svc.cluster.local -p 5432; do
                    echo "waiting for postgres..."
                    sleep 2
                  done
              envFrom:
                - configMapRef:
                    name: geo-api-config
                - secretRef:
                    name: geo-api-secret
              resources:
                requests:
                  memory: "128Mi"
                  cpu: "50m"
                limits:
                  memory: "256Mi"
                  cpu: "200m"

            # 2. Run Alembic migrations (DEPLOY-04, D-13)
            - name: alembic-migrate
              image: ghcr.io/civicpulse/geo-api:latest
              command: ["alembic", "upgrade", "head"]
              envFrom:
                - configMapRef:
                    name: geo-api-config
                - secretRef:
                    name: geo-api-secret
              resources:
                requests:
                  memory: "128Mi"
                  cpu: "50m"
                limits:
                  memory: "256Mi"
                  cpu: "200m"

            # 3. Rebuild spell dictionary (DEPLOY-04, RESIL-04, D-13)
            - name: spell-rebuild
              image: ghcr.io/civicpulse/geo-api:latest
              command: ["geo-import", "rebuild-dictionary"]
              envFrom:
                - configMapRef:
                    name: geo-api-config
                - secretRef:
                    name: geo-api-secret
              resources:
                requests:
                  memory: "128Mi"
                  cpu: "50m"
                limits:
                  memory: "256Mi"
                  cpu: "200m"

            # 4. Pull Ollama model if not cached on PVC (D-07)
            - name: model-pull
              image: ollama/ollama:latest
              command: ["sh", "-c"]
              args:
                - |
                  /bin/ollama serve &
                  until curl -sf http://localhost:11434/api/tags > /dev/null 2>&1; do sleep 2; done
                  if ! ollama list | grep -q "qwen2.5:3b"; then
                    ollama pull qwen2.5:3b
                  fi
                  kill %1
              volumeMounts:
                - name: ollama-data
                  mountPath: /root/.ollama
              resources:
                requests:
                  memory: "512Mi"
                  cpu: "100m"
                limits:
                  memory: "1Gi"
                  cpu: "500m"

            # 5. Ollama sidecar -- MUST BE LAST (D-08, native sidecar K8s 1.29+)
            - name: ollama
              image: ollama/ollama:latest
              restartPolicy: Always
              ports:
                - containerPort: 11434
              volumeMounts:
                - name: ollama-data
                  mountPath: /root/.ollama
              startupProbe:
                httpGet:
                  path: /api/tags
                  port: 11434
                failureThreshold: 30
                periodSeconds: 10
              readinessProbe:
                httpGet:
                  path: /api/tags
                  port: 11434
                initialDelaySeconds: 5
                periodSeconds: 10
                failureThreshold: 6
              livenessProbe:
                httpGet:
                  path: /api/tags
                  port: 11434
                initialDelaySeconds: 30
                periodSeconds: 30
                failureThreshold: 3
              resources:
                requests:
                  memory: "2Gi"
                  cpu: "500m"
                limits:
                  memory: "4Gi"

          containers:
            - name: geo-api
              image: ghcr.io/civicpulse/geo-api:latest
              ports:
                - containerPort: 8000
              envFrom:
                - configMapRef:
                    name: geo-api-config
                - secretRef:
                    name: geo-api-secret
              startupProbe:
                httpGet:
                  path: /health/live
                  port: 8000
                failureThreshold: 30
                periodSeconds: 10
              livenessProbe:
                httpGet:
                  path: /health/live
                  port: 8000
                periodSeconds: 30
                timeoutSeconds: 5
                failureThreshold: 3
              readinessProbe:
                httpGet:
                  path: /health/ready
                  port: 8000
                periodSeconds: 10
                timeoutSeconds: 5
                failureThreshold: 3
              lifecycle:
                preStop:
                  exec:
                    command: ["/bin/sleep", "10"]
              resources:
                requests:
                  memory: "256Mi"
                  cpu: "100m"
                limits:
                  memory: "512Mi"
                  cpu: "500m"

          volumes:
            - name: ollama-data
              persistentVolumeClaim:
                claimName: ollama-pvc
    ```

    **Critical ordering rules:**
    - db-wait -> alembic-migrate -> spell-rebuild -> model-pull -> ollama (sidecar LAST)
    - The `ollama` initContainer has `restartPolicy: Always` making it a native K8s sidecar
    - geo-api main container probes use /health/live (liveness) and /health/ready (readiness) from Plan 01
    - preStop sleep(10) per D-11, terminationGracePeriodSeconds: 30 per D-11

    After creating the deployment.yaml, validate the full base builds:

