<objective>
Fix the silent bug where `set_official()` never writes to the `admin_overrides` table, and document the GIS-first import ordering constraint.

Purpose: GEO-07 requires admin custom coordinates to persist in `admin_overrides`. Without this write, the CLI import guard (which checks `admin_overrides` to decide whether to auto-set official) never finds a row, so admin overrides can be silently displaced by a subsequent GIS import. DATA-03 compliance depends on import ordering; this must be documented.

Output: Fixed `geocoding.py` with AdminOverride upsert, updated tests, documented CLI constraint.
</objective>

```python
    def test_import_skips_official_when_admin_override_exists(self):
        """When admin_overrides row exists for an address, CLI does NOT insert into official_geocoding."""
        from typer.testing import CliRunner
        from civpulse_geo.cli import app

        runner = CliRunner()
        mock_engine = MagicMock()
        mock_conn = MagicMock()
        mock_engine.return_value.connect.return_value.__enter__ = MagicMock(return_value=mock_conn)
        mock_engine.return_value.connect.return_value.__exit__ = MagicMock(return_value=False)

        # Track call sequence to distinguish address upsert, geocoding upsert,
        # admin_overrides check, and official_geocoding insert
        call_count = {"n": 0}
        def execute_side_effect(stmt, params=None):
            call_count["n"] += 1
            result = MagicMock()

            stmt_str = str(stmt) if not isinstance(stmt, str) else stmt
            sql_text = getattr(stmt, 'text', str(stmt))

            if "admin_overrides" in sql_text:
                # Return a row indicating admin override EXISTS
                result.fetchone.return_value = (99,)
                return result
            elif "official_geocoding" in sql_text:
                # This should NOT be reached when admin override exists
                pytest.fail("official_geocoding INSERT should not execute when admin_overrides row exists")
            else:
                # Address upsert or geocoding_results upsert
                result.fetchone.return_value = (1, True)
                return result

        mock_conn.execute.side_effect = execute_side_effect

        with patch("civpulse_geo.cli.create_engine", mock_engine):
            result = runner.invoke(
                app,
                [str(GEOJSON_PATH), "--database-url", "postgresql+psycopg2://test:test@localhost/test"],
            )
            assert result.exit_code == 0, f"CLI failed: {result.output}\n{result.exception}"
```

This test works by making the `admin_overrides` SELECT return a row (id=99), which causes the `if override_row is None:` guard to skip the official_geocoding INSERT. If the official INSERT is ever reached, `pytest.fail` fires.
