    **similarity_to_confidence() function:**
    ```python
    def similarity_to_confidence(similarity: float) -> float:
        """Map word_similarity 0.65-1.0 to confidence 0.50-0.75 (D-07).
        Linear: confidence = 0.50 + (similarity - 0.65) / 0.35 * 0.25"""
        normalized = (similarity - FUZZY_SIMILARITY_THRESHOLD) / (1.0 - FUZZY_SIMILARITY_THRESHOLD)
        return FUZZY_CONFIDENCE_MIN + normalized * (FUZZY_CONFIDENCE_MAX - FUZZY_CONFIDENCE_MIN)
    ```

    **FuzzyMatcher class:**
    ```python
    from sqlalchemy.ext.asyncio import AsyncSession, async_sessionmaker
    from sqlalchemy import func, select, text, union_all, literal_column

    class FuzzyMatcher:
        def __init__(self, session_factory: async_sessionmaker[AsyncSession]) -> None:
            self._session_factory = session_factory

        async def find_fuzzy_match(
            self,
            street_name: str,
            zip_code: str | None = None,
            street_number: str | None = None,
        ) -> FuzzyMatchResult | None:
            """Find best fuzzy match across OA, NAD, Macon-Bibb staging tables.

            Returns single best candidate (D-14) or None if nothing above threshold.
            Uses word_similarity() with dmetaphone() tiebreaker (D-12).
            """
    ```

    The `find_fuzzy_match` method:
    1. Builds three SELECT queries (OA, NAD, Macon-Bibb) each selecting street_name, street_number, city, zip/postcode, lat/lng from ST_Y/ST_X on location, word_similarity score, and a literal source tag
    2. Each query filters by `word_similarity(input_street, table.street_name) >= 0.65` and optionally by zip_code (if provided) using LIKE prefix match for zip flexibility
    3. UNION ALL the three queries, ORDER BY score DESC, LIMIT 5
    4. If only one candidate or top candidate leads by > 0.05 over second: return top candidate
    5. If multiple within 0.05: execute dmetaphone tiebreaker — compare `dmetaphone(upper(input_street))` with `dmetaphone(upper(candidate.street_name))` for each, prefer phonetic match, then highest score
    6. Map score to confidence via similarity_to_confidence()
    7. Return FuzzyMatchResult or None

    **word_similarity() argument order:** Always `word_similarity(:input, table.street_name)` — input (needle) first, stored (haystack) second. This is critical per Research Pitfall 1.

    **Zip filtering:** For OA use `OpenAddressesPoint.postcode`, for NAD use `NADPoint.zip_code`, for Macon-Bibb use `MaconBibbPoint.zip_code`. Use LIKE prefix match (`zip_code LIKE :prefix%`) to handle truncated zips from Phase 12 fix.

    **dmetaphone tiebreaker SQL pattern:**
    Use `func.dmetaphone(func.upper(literal_column(...)))` for both input and candidate. Compare primary codes first, then alt codes via `dmetaphone_alt()`. Phonetic match = True wins over False. If still tied, highest score wins.
