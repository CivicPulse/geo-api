## Cross-Phase Integration

**Score: 23/25 requirement wiring verified** | 4 defects (1 critical, 1 medium, 1 low, 1 info)

### Critical: FuzzyMatcher Never Registered at Startup

**File:** `src/civpulse_geo/main.py`
**Affected:** FUZZ-02, FUZZ-03, FUZZ-04, CASC-01 (stage 3)

`FuzzyMatcher` (Phase 13-02) is consumed by `CascadeOrchestrator` stage 3. The API routes read `getattr(request.app.state, "fuzzy_matcher", None)`. But `main.py` never imports, instantiates, or assigns `FuzzyMatcher` to `app.state`.

**Result:** `fuzzy_matcher` is always `None`. Stage 3 guard (`if not skip_fuzzy and fuzzy_matcher is not None:`) never enters. Fuzzy matching silently never fires.

**Fix:**
```python
# In main.py lifespan, after spell corrector:
from civpulse_geo.services.fuzzy import FuzzyMatcher
app.state.fuzzy_matcher = FuzzyMatcher(AsyncSessionLocal)
```

### Medium: Legacy Path 3-Tuple Unpack Crash

**File:** `src/civpulse_geo/services/geocoding.py:214`
**Affected:** FIX-02 (legacy path stability)

`_legacy_geocode` unpacks `_parse_input_address(normalized)` as 3-tuple but the function returns a 5-tuple since Phase 12. Raises `ValueError` when `CASCADE_ENABLED=false`.

**Fix:** `street_number, street_name, postal_code, _, _ = _parse_input_address(normalized)`

### Low: Cascade Path Always Reports cache_hit=False

**File:** `src/civpulse_geo/services/cascade.py:840`
**Affected:** CASC-01

`CascadeOrchestrator.run()` hardcodes `cache_hit=False` — no cache-check phase exists. Repeated calls to the same address re-dispatch all providers every time. Behavioral regression vs v1.1 which had a full cache-hit path.

### Info: No geo-api K8s Deployment Manifest

**Affected:** LLM-01, LLM-04

Ollama K8s manifests (`ollama-deployment.yaml`, `ollama-pvc.yaml`, `ollama-service.yaml`) are complete, but no geo-api K8s Deployment manifest exists to inject `OLLAMA_URL` or `CASCADE_LLM_ENABLED` env vars for production K8s deployment.
