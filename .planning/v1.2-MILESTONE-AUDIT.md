## Cross-Phase Integration

**Score: 23/25 requirement wiring verified**

### Critical Defects

#### 1. FuzzyMatcher never registered at startup

**Severity:** Critical
**Affected:** FUZZ-02, FUZZ-03, FUZZ-04, CASC-01 (stage 3)
**File:** `src/civpulse_geo/main.py`

`FuzzyMatcher` is created by Phase 13-02 and consumed by Phase 14 `CascadeOrchestrator` stage 3. The API routes read `app.state.fuzzy_matcher` via `getattr(request.app.state, "fuzzy_matcher", None)`. However, `main.py`'s lifespan function never imports, instantiates, or assigns `FuzzyMatcher` to `app.state`.

**Result:** `fuzzy_matcher` is always `None`. Stage 3 guard `if not skip_fuzzy and fuzzy_matcher is not None:` never enters. Addresses that fail exact match but would succeed on fuzzy match silently get no fuzzy candidates.

**Fix:** Add to `main.py` lifespan after spell corrector section:
```python
from civpulse_geo.services.fuzzy import FuzzyMatcher
app.state.fuzzy_matcher = FuzzyMatcher(AsyncSessionLocal)
```

#### 2. Legacy path 3-tuple unpack crash

**Severity:** Medium
**Affected:** FIX-02 (legacy path stability)
**File:** `src/civpulse_geo/services/geocoding.py` line 214

`_legacy_geocode` unpacks `_parse_input_address(normalized)` as `street_number, street_name, postal_code` (3 values) but the function returns a 5-tuple since Phase 12-01. This raises `ValueError: too many values to unpack` when `CASCADE_ENABLED=false` AND all local providers return NO_MATCH.

**Impact:** Low in practice (cascade defaults to enabled), but crashes the legacy fallback path entirely.

**Fix:** Change to `street_number, street_name, postal_code, _, _ = _parse_input_address(normalized)`
