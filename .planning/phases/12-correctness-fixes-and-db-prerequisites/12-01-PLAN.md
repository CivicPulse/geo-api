<objective>
Fix street suffix matching (FIX-03) and add zip prefix fallback (FIX-02) across all three local providers (OA, NAD, Macon-Bibb).

Purpose: Multi-word streets like "Beaver Falls" currently fail because scourgify normalizes the suffix away and the query only matches on StreetName. Truncated zips (e.g., "3120") return NO_MATCH instead of falling back to prefix matching. Both bugs corrupt cascade results in Phase 14.

Output: Updated `_parse_input_address()` returning 5-tuple, suffix-aware match functions, zip prefix fallback functions, and comprehensive tests.
</objective>
