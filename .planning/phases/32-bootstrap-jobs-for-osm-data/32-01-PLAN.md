**pbf-download-job.yaml** (JOB-01):
- `image: curlimages/curl:8`
- `spec.backoffLimit: 2`
- `spec.activeDeadlineSeconds: 900`
- resources: requests 100m CPU / 256Mi mem; limits 500m / 512Mi
- Shell-guard + download (ensure target dir exists; curl fails fast on HTTP errors):
```
set -eu
mkdir -p /data/pbf
TARGET=/data/pbf/georgia-latest.osm.pbf
if [ -f "$TARGET" ]; then
  SIZE=$(stat -c%s "$TARGET" 2>/dev/null || stat -f%z "$TARGET")
  if [ "$SIZE" -gt 104857600 ]; then
    echo "PBF already present (size=$SIZE bytes), skipping download"
    exit 0
  fi
  echo "PBF exists but is too small ($SIZE bytes), re-downloading"
  rm -f "$TARGET"
fi
echo "Downloading Georgia PBF..."
curl -fsSL --retry 3 --retry-delay 10 -o "$TARGET" https://download.geofabrik.de/north-america/us/georgia-latest.osm.pbf
echo "Download complete: $(stat -c%s $TARGET 2>/dev/null || stat -f%z $TARGET) bytes"
```
- volumes: `osm-pbf` -> PVC `osm-pbf-pvc`; volumeMount at `/data/pbf`

**tile-import-job.yaml** (JOB-03):
- `image: overv/openstreetmap-tile-server:2.3.0`
- `spec.backoffLimit: 1`
- `spec.activeDeadlineSeconds: 3600`
- resources: requests 1 CPU / 4Gi mem; limits 2 CPU / 8Gi mem
- Shell-guard: tile-server import creates a postgres DB inside osm-tile-data-pvc at /data/database.
  Use marker-file approach since probing PG role requires starting PG. Use an import-complete
  marker the image creates implicitly (the presence of the `PG_VERSION` file in the postgres data
  dir indicates import already ran):
```
set -eu
if [ -f /data/database/postgres/PG_VERSION ]; then
  echo "tile-server data already imported (postgres dir present), skipping"
  exit 0
fi
echo "Running tile-server import..."
# Image uses /run.sh as its entrypoint wrapper; re-invoke with import arg.
exec /run.sh import
```
- volumes:
    `osm-tile-data` -> PVC `osm-tile-data-pvc` (mounted at /data/database)
    `osm-pbf` -> PVC `osm-pbf-pvc` readOnly: true
- volumeMounts:
    osm-tile-data at /data/database
    osm-pbf at /data/region.osm.pbf subPath: pbf/georgia-latest.osm.pbf readOnly: true
- env (from Phase 24 compose): `NAME_LUA=openstreetmap-carto.lua` is default; no extras needed

**valhalla-build-job.yaml** (JOB-04):
- `image: ghcr.io/valhalla/valhalla:latest`
- `spec.backoffLimit: 1`
- `spec.activeDeadlineSeconds: 3600`
- resources: requests 2 CPU / 8Gi mem; limits 4 CPU / 12Gi mem
- env (force a one-shot build, don't serve):
    serve_tiles: "False"
    force_rebuild: "True"
    build_admins: "False"
    build_elevation: "False"
    use_tiles_ignore_pbf: "False"
    tile_urls: ""
- Shell-guard (Valhalla entrypoint runs `valhalla_build_tiles` then exits when serve_tiles=False):
```
set -eu
if [ -d /custom_files/valhalla_tiles ] && [ "$(ls -A /custom_files/valhalla_tiles 2>/dev/null)" ]; then
  echo "Valhalla tiles already built, skipping"
  exit 0
fi
if [ ! -f /custom_files/georgia-latest.osm.pbf ]; then
  echo "Copying PBF into valhalla custom_files/ (entrypoint expects it there)..."
  cp /data/pbf/georgia-latest.osm.pbf /custom_files/georgia-latest.osm.pbf
fi
echo "Starting Valhalla build..."
exec /valhalla/scripts/run.sh
```
- volumes:
    `valhalla-tiles` -> PVC `valhalla-tiles-pvc` (mounted at /custom_files)
    `osm-pbf` -> PVC `osm-pbf-pvc` readOnly: true (mounted at /data/pbf)

Implements requirements JOB-01, JOB-03, JOB-04, and JOB-05 (ArgoCD sync hook
annotations make Jobs trigger on sync per D-03).
