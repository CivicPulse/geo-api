2. `Cluster` — used in consensus scoring:
   ```python
   @dataclass
   class Cluster:
       members: list[ProviderCandidate] = field(default_factory=list)
       centroid_lat: float = 0.0
       centroid_lng: float = 0.0
       total_weight: float = 0.0

       def add(self, candidate: ProviderCandidate) -> None:
           self.members.append(candidate)
           self.total_weight += candidate.weight
           # Recompute weighted centroid (D-07)
           sum_w = sum(m.weight for m in self.members)
           self.centroid_lat = sum(m.lat * m.weight for m in self.members) / sum_w
           self.centroid_lng = sum(m.lng * m.weight for m in self.members) / sum_w

       @classmethod
       def from_seed(cls, candidate: ProviderCandidate) -> Cluster:
           c = cls()
           c.add(candidate)
           return c
   ```

**CascadeOrchestrator class:**

```python
class CascadeOrchestrator:
    """Staged cascade resolution pipeline (CASC-01).

    Stages: normalize → spell-correct → exact match → fuzzy → consensus → auto-set
    """

    async def run(
        self,
        freeform: str,
        db: AsyncSession,
        providers: dict[str, GeocodingProvider],
        http_client: httpx.AsyncClient,
        spell_corrector: SpellCorrector | None = None,
        fuzzy_matcher: FuzzyMatcher | None = None,
        dry_run: bool = False,
        trace: bool = False,
    ) -> CascadeResult:
```

**Stage implementation inside run():**

**Stage 1 — Normalize + spell-correct:**
- Apply spell correction if spell_corrector is not None (reuse GeocodingService._apply_spell_correction logic inline)
- Call canonical_key() to get (normalized, address_hash)
- Find or create Address record (same pattern as current geocode())
- Record trace entry if trace or dry_run

**Stage 2 — Exact match (all providers in parallel, D-05):**
- Partition providers into local and remote using `provider.is_local`
- Use `asyncio.gather(*tasks, return_exceptions=True)` with `asyncio.wait_for(timeout=settings.exact_match_timeout_ms / 1000)` per provider call
- Collect all successful GeocodingResultSchema objects
- For remote providers that succeed, upsert into geocoding_results table (same pattern as current geocode() Step 5)
- Build `ProviderCandidate` list from all results (local and remote)
- Record trace entry

**Early-exit check (D-12):**
- If ANY candidate has confidence >= 0.80, set `skip_fuzzy = True`
- Consensus scoring ALWAYS runs regardless (D-13)

**Stage 3 — Fuzzy match (if not skip_fuzzy, D-14):**
- Extract street_name, zip_code, street_number from normalized address via `_parse_input_address()`
- Call `fuzzy_matcher.find_fuzzy_match(street_name, zip_code, street_number)` with `asyncio.wait_for(timeout=settings.fuzzy_match_timeout_ms / 1000)`
- If result returned and result.lat is not None, create ProviderCandidate with:
  - `provider_name = f"fuzzy_{result.source}"` (e.g., "fuzzy_openaddresses")
  - `weight = get_provider_weight(result.source) * (result.confidence / 0.80)` (D-09)
  - `is_fuzzy = True`
- Add to candidates list
- Record trace entry

**Stage 4 — Consensus scoring (CONS-01):**
- Call `run_consensus(candidates)`
- If winning cluster exists:
  - Flag outlier_providers set from candidates where is_outlier is True
  - Determine set_by_stage: "exact_match_consensus" if no fuzzy members in winning cluster, "fuzzy_consensus" if winning cluster has fuzzy members, "single_provider" if only 1 candidate total
- Record trace entry

**Stage 5 — Auto-set official (CONS-04):**
- Single-result handling (D-11): if only 1 candidate total and confidence < 0.80, skip auto-set
- Check for existing admin override: query OfficialGeocoding joined to GeocodingResult where provider_name == "admin_override", skip if exists (D-22). Also check AdminOverride table.
- For the winning candidate:
  - If it came from a remote provider, its ORM row already exists from Stage 2 upsert. Use that geocoding_result_id.
  - If it came from a local or fuzzy provider, persist a new GeocodingResult ORM row first (upsert into geocoding_results with the candidate's lat/lng/confidence/provider_name), then use the returned id.
- If dry_run: populate `would_set_official` on CascadeResult, do NOT write OfficialGeocoding
- If not dry_run: upsert OfficialGeocoding with `on_conflict_do_update` setting geocoding_result_id and set_by_stage (D-22)
- Record trace entry

**Stage 6 — Commit and return:**
- `await db.commit()`
- Build and return CascadeResult with all fields populated
- If trace or dry_run, populate cascade_trace list

**Timeout handling (D-16):** Wrap the entire run() in a try with asyncio.wait_for(settings.cascade_total_timeout_ms / 1000). On timeout, return whatever results are available so far (graceful degradation).

**Key anti-patterns to AVOID:**
- Do NOT pass db session to FuzzyMatcher — it uses its own session_factory (see anti-patterns in RESEARCH.md)
- Do NOT assume local results have geocoding_result_id — they don't, must persist first before OfficialGeocoding write
- Do NOT use `on_conflict_do_nothing` — cascade uses `on_conflict_do_update` (D-22)
- Do NOT skip consensus when early-exit fires (D-13)
