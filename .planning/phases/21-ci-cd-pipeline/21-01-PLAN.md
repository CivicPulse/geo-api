2. Create `.github/workflows/ci.yml` with the following exact content structure:

```yaml
name: CI

on:
  pull_request:
    branches: [main]

permissions:
  contents: read

jobs:
  lint-and-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd # v6.0.2

      - name: Lint with ruff
        uses: astral-sh/ruff-action@4919ec5cf1f49eff0871dbcea0da843445b837e6 # v3.6.1
        with:
          src: "src"

      - name: Set up uv and Python 3.12
        uses: astral-sh/setup-uv@cec208311dfd045dd5311c1add060b2062131d57 # v8.0.0
        with:
          python-version: "3.12"
          enable-cache: true
          cache-dependency-glob: "uv.lock"

      - name: Install dependencies
        run: uv sync --locked --frozen

      - name: Run test suite
        run: uv run pytest
```

Key requirements per locked decisions:
- Per D-07: This is the `ci.yml` workflow, triggered on `pull_request` to `main` only.
- Per D-08: `permissions: contents: read` -- minimal permissions, CI needs only read access.
- Per D-09: ALL `uses:` lines are pinned to full 40-character commit SHAs with version comments. The exact SHAs from the Research document's "Verified Action Commit SHAs" table:
  - `actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd` (v6.0.2)
  - `astral-sh/ruff-action@4919ec5cf1f49eff0871dbcea0da843445b837e6` (v3.6.1)
  - `astral-sh/setup-uv@cec208311dfd045dd5311c1add060b2062131d57` (v8.0.0)
- `ruff-action` with `src: "src"` scopes lint to the source directory.
- `setup-uv` with `enable-cache: true` and `cache-dependency-glob: "uv.lock"` for cache keyed on lockfile hash.
- `uv sync --locked --frozen` installs all deps (including dev deps like pytest, pytest-asyncio).
- `uv run pytest` uses testpaths from pyproject.toml (`tests/`). No path argument needed.
- No database or system libraries required -- all 548 tests use mocked dependencies. Fiona bundles its own GDAL.
- Job name `lint-and-test` is used as the required status check name.

3. Run `uv run pytest tests/ -x -q` locally to confirm the test suite passes (pre-existing validation).
