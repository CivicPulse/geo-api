  <action>
    This is a diagnostic task. No repo files are modified. All commands run against the live cluster.

    **Tempo Diagnosis**

    1. Check if Tempo is deployed and running in civpulse-infra:
       ```
       kubectl get pods -n civpulse-infra -l app=tempo 2>/dev/null || kubectl get pods -n civpulse-infra | grep -i tempo
       kubectl get svc -n civpulse-infra | grep -i tempo
       ```
       If Tempo pod is not running or service does not exist, that is the root cause.

    2. Check if the geo-api pod is actually receiving the ConfigMap env var:
       ```
       kubectl exec -n civpulse-prod deploy/geo-api -c geo-api -- env | grep -i otel
       ```
       Expected: OTEL_EXPORTER_ENDPOINT=http://tempo.civpulse-infra.svc.cluster.local:4317
       If it shows http://tempo:4317 (the code default), the ConfigMap is not being applied.

    3. Test gRPC connectivity from the geo-api pod to Tempo:
       ```
       kubectl exec -n civpulse-prod deploy/geo-api -c geo-api -- python -c "
       import socket
       try:
           host = 'tempo.civpulse-infra.svc.cluster.local'
           sock = socket.create_connection((host, 4317), timeout=5)
           print(f'Connected to {host}:4317 OK')
           sock.close()
       except Exception as e:
           print(f'Connection failed: {e}')
       "
       ```

    4. If Tempo service exists but pod is not running, check why:
       ```
       kubectl describe pod -n civpulse-infra -l app=tempo 2>/dev/null | tail -30
       ```

    5. If Tempo is not deployed at all, document this as a prerequisite — Tempo must be deployed to civpulse-infra namespace for OTel traces to work. Check if there is a Tempo deployment manifest or Helm chart in the infra repo.

    **Tiger Diagnosis**

    6. Check if postgis_tiger_geocoder extension exists in the prod database:
       ```
       kubectl port-forward svc/postgresql -n civpulse-infra 5432:5432 &
       sleep 2
       PGPASSWORD=L7mYpqUwn6BVx7X8R6xhWJxsGDF6x3S psql -h localhost -U civpulse_geo_prod -d civpulse_geo_prod -c "SELECT * FROM pg_available_extensions WHERE name = 'postgis_tiger_geocoder';"
       ```

    7. If the extension is available but not installed:
       ```
       PGPASSWORD=L7mYpqUwn6BVx7X8R6xhWJxsGDF6x3S psql -h localhost -U civpulse_geo_prod -d civpulse_geo_prod -c "CREATE EXTENSION IF NOT EXISTS postgis_tiger_geocoder CASCADE;"
       ```

    8. If the extension is NOT available (not in pg_available_extensions), Tiger geocoding requires the Tiger census data to be loaded AND the extension to be compiled into the PostgreSQL server. This is typically a DBA/infra task. Document the finding.

    9. Kill the port-forward:
       ```
       kill %1
       ```

    **Document findings** for the checkpoint task.
