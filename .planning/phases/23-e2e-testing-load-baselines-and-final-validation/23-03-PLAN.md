    2. Create `scripts/verify/verify_loki.py` (per D-10, D-11a, D-12, TEST-04):
       - Shebang: `#!/usr/bin/env python3`
       - Module docstring explaining purpose, required port-forward command (`kubectl port-forward -n civpulse-infra svc/loki 3100:3100`), and expected output.
       - Import `sys`, `json`, `time`, `os`, `httpx`.
       - Define `LOKI_URL = os.environ.get("LOKI_URL", "http://localhost:3100")`.
       - Define `LOOKBACK_MINUTES = int(os.environ.get("VERIFY_LOOKBACK_MINUTES", "30"))`.
       - Function `verify_loki_logs()`:
         - Calculate time window: `end_ns = int(time.time() * 1e9)`, `start_ns = int((time.time() - LOOKBACK_MINUTES * 60) * 1e9)`.
         - Build LogQL query: `'{namespace="civpulse-prod"} | json | request_id != ""'`
           (Use namespace label selector per Pitfall 6 — the `service` label may not match Alloy's label config. Filtering on namespace + JSON parsing for `request_id` is more reliable.)
         - GET `{LOKI_URL}/loki/api/v1/query_range` with params: `query`, `start=str(start_ns)`, `end=str(end_ns)`, `limit=100`.
         - Check `r.status_code == 200`, print error and `sys.exit(1)` if not.
         - Parse `results = r.json()["data"]["result"]`.
         - If `not results`: print `"FAIL: No logs found in Loki for namespace=civpulse-prod with request_id"` to stderr, `sys.exit(1)`.
         - Count total entries across all streams.
         - For each stream in results, for each `(ts, line)` in stream `values`:
           - `entry = json.loads(line)`
           - Check `"request_id" in entry` — if missing, print FAIL with the entry, `sys.exit(1)`.
           - Check `"trace_id" in entry` — if missing, print FAIL with the entry, `sys.exit(1)`.
         - Track count of entries with both fields.
         - Print `f"PASS: {count} log entries verified — all have request_id and trace_id"`.
       - `if __name__ == "__main__": verify_loki_logs()`

    3. Create `scripts/verify/verify_tempo.py` (per D-10, D-11b, D-12, TEST-05):
       - Shebang: `#!/usr/bin/env python3`
       - Module docstring explaining purpose, required port-forward command (`kubectl port-forward -n civpulse-infra svc/tempo 3200:3200`), and expected output.
       - Import `sys`, `time`, `os`, `httpx`.
       - Define `TEMPO_URL = os.environ.get("TEMPO_URL", "http://localhost:3200")`.
       - Define `LOOKBACK_MINUTES = int(os.environ.get("VERIFY_LOOKBACK_MINUTES", "30"))`.
       - Function `verify_tempo_traces()`:
         - Calculate time window: `end_s = int(time.time())`, `start_s = int(time.time() - LOOKBACK_MINUTES * 60)`.
         - GET `{TEMPO_URL}/api/search` with params: `tags="service.name=civpulse-geo"`, `start=str(start_s)`, `end=str(end_s)`, `limit=20`.
         - Check `r.status_code == 200`.
         - Parse `traces = r.json().get("traces", [])`.
         - If `not traces`: print `"FAIL: No traces found in Tempo for service.name=civpulse-geo"` to stderr, `sys.exit(1)`.
         - Print `f"Found {len(traces)} traces in Tempo"`.
         - Pick the first trace, get its `traceID`.
         - GET `{TEMPO_URL}/api/traces/{trace_id}` to fetch full trace detail.
         - Parse the OTLP response. Extract span names from the nested batches structure:
           ```python
           spans = []
           for batch in r.json().get("batches", []):
               for scope_spans in batch.get("scopeSpans", []):
                   for span in scope_spans.get("spans", []):
                       spans.append(span)
           ```
         - If `len(spans) < 2`: print `"FAIL: Trace {trace_id} has fewer than 2 spans — expected parent + child spans"`, `sys.exit(1)`.
         - Check for DB spans: look for any span with an attribute where key is `db.system`. Extract attributes from span `attributes` list (OTLP format: `[{"key": "db.system", "value": {"stringValue": "postgresql"}}]`).
         - If no DB span found: print `"WARN: No db.system span attribute found — DB auto-instrumentation may not be active"` (warning, not failure — some traces may not hit DB).
         - Print `f"PASS: {len(traces)} traces found, sample trace {trace_id} has {len(spans)} spans"`.
       - `if __name__ == "__main__": verify_tempo_traces()`
  </action>

      `kubectl port-forward -n civpulse-infra pod/victoria-metrics-victoria-metrics-single-server-0 8428:8428`
      (Note: must port-forward to the POD, not the Service — VictoriaMetrics uses a headless Service with ClusterIP: None, per Pitfall 4).
    - Import `sys`, `os`, `httpx`.
    - Define `VM_URL = os.environ.get("VICTORIAMETRICS_URL", "http://localhost:8428")`.
    - Helper function `query_vm(promql: str) -> dict`:
      - GET `{VM_URL}/api/v1/query` with params `{"query": promql}`.
      - Check `r.status_code == 200`, return `r.json()`.
    - Helper function `assert_metric_exists(name: str, promql: str) -> int`:
      - Call `query_vm(promql)`.
      - Parse `result = data["data"]["result"]`.
      - If `not result`: print `f"FAIL: Metric '{name}' returned no results for query: {promql}"` to stderr, `sys.exit(1)`.
      - Return `len(result)` (number of time series).
    - Function `verify_victoriametrics()`:
      - **Check 1: Request rate metric (http_requests_total)**
        - Query: `'http_requests_total{}'` (no job filter initially — the job label depends on VictoriaMetrics scrape config. If no results, try with `job="geo-api"` as fallback.)
        - Call `assert_metric_exists("http_requests_total", "http_requests_total")`.
        - Print count of time series found.
      - **Check 2: Latency histogram (http_request_duration_seconds)**
        - Query: `'http_request_duration_seconds_bucket'`
        - Call `assert_metric_exists("http_request_duration_seconds_bucket", "http_request_duration_seconds_bucket")`.
        - Also query `'histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m]))'` and print the P95 value if available (informational, not a pass/fail gate — rate may be 0 if no recent requests).
      - **Check 3: Provider requests metric (geo_provider_requests_total)**
        - Query: `'geo_provider_requests_total'`
        - Call `assert_metric_exists("geo_provider_requests_total", "geo_provider_requests_total")`.
        - Print count of time series found and list the provider label values.
      - **Check 4: Error rate sanity**
        - Query: `'http_requests_total{status_code=~"5.."}'` to check for 5xx errors.
        - This is informational only — print count but do NOT fail. Some 5xx may be expected during load tests.
      - Print `"PASS: All 3 core metric families verified — http_requests_total, http_request_duration_seconds, geo_provider_requests_total"`.
    - `if __name__ == "__main__": verify_victoriametrics()`
