Output: `loadtests/` directory with Locust task file, address data files for cold and warm runs, and a gitignored reports directory.

    ```python
    Cold-cache run (unique addresses):
      locust -f loadtests/geo_api_locustfile.py --headless --users 30 --spawn-rate 0.25 --run-time 7m --host http://localhost:8000 --csv loadtests/reports/cold_cache --csv-full-history

    Warm-cache run (repeated addresses):
      GEO_LOADTEST_WARM=1 locust -f loadtests/geo_api_locustfile.py --headless --users 30 --spawn-rate 0.25 --run-time 7m --host http://localhost:8000 --csv loadtests/reports/warm_cache --csv-full-history
    """
    ```

    Implementation details:
    - Import `os`, `random`, `pathlib` from stdlib; `HttpUser`, `task`, `between` from `locust`.
    - At module level, resolve the address file directory using `pathlib.Path(__file__).parent / "addresses"`.
    - Load address list based on `GEO_LOADTEST_WARM` env var:
      - If `os.environ.get("GEO_LOADTEST_WARM")` is truthy: load from `warm_cache_addresses.txt`
      - Otherwise: load from `cold_cache_addresses.txt`
    - Read file into a list of stripped, non-empty lines.
    - Define `class GeoApiUser(HttpUser):` with:
      - `wait_time = between(0.5, 2.0)`
      - `@task(5)` for `def geocode(self)`: picks a random address from the loaded list, calls `self.client.post("/geocode", json={"address": addr})`.
      - `@task(3)` for `def validate(self)`: picks a random address, calls `self.client.post("/validate", json={"address": addr})`.
      - `@task(2)` for `def cascade_trace(self)`: picks a random address, calls `self.client.post("/geocode", json={"address": addr}, params={"trace": "true"})`.
    - Task weights: 50% geocode, 30% validate, 20% cascade — per Claude's discretion for reasonable endpoint distribution (D-08 specifies these three endpoints).
    - The `--spawn-rate 0.25` CLI flag handles D-09's 2-minute ramp-up (30 users / 120 seconds = 0.25 users/sec). This is NOT set in Python code — it's a CLI parameter.
    - Add a module-level docstring with the exact CLI commands for cold-cache and warm-cache runs (shown above).
