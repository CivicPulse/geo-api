  <action>
Build and push the Docker image to GHCR per D-08, D-09, D-10.

**Step 1: Authenticate to GHCR** using existing gh CLI auth (confirmed `write:packages` scope):
```bash
echo $(gh auth token) | docker login ghcr.io -u kerryhatcher --password-stdin
```

**Step 2: Determine git SHA for tagging:**
```bash
SHORT_SHA=$(git rev-parse --short HEAD)
```

**Step 3: Build the image** with both sha tag and latest per D-09:
```bash
docker build \
  --build-arg GIT_COMMIT=${SHORT_SHA} \
  -t ghcr.io/civicpulse/geo-api:sha-${SHORT_SHA} \
  -t ghcr.io/civicpulse/geo-api:latest \
  .
```

**Step 4: Verify build** before pushing:
- `docker run --rm ghcr.io/civicpulse/geo-api:sha-${SHORT_SHA} id` -- must show `uid=1000(appuser)`
- `docker run --rm ghcr.io/civicpulse/geo-api:sha-${SHORT_SHA} python -c "import fiona; print('fiona OK')"` -- must print `fiona OK`
- `docker run --rm ghcr.io/civicpulse/geo-api:sha-${SHORT_SHA} python -c "from civpulse_geo.main import app; print('app OK')"` -- must print `app OK`

**Step 5: Push to GHCR:**
```bash
docker push ghcr.io/civicpulse/geo-api:sha-${SHORT_SHA}
docker push ghcr.io/civicpulse/geo-api:latest
```

**Step 6: Verify k3s can pull** by running a short-lived pod in civpulse-dev:
```bash
kubectl run ghcr-pull-test --rm -i --restart=Never \
  --namespace civpulse-dev \
  --image=ghcr.io/civicpulse/geo-api:sha-${SHORT_SHA} \
  -- id
```
Expected output: `uid=1000(appuser) gid=1000(appuser) groups=1000(appuser)`

If the image is not publicly visible (ErrImagePull), the user must go to https://github.com/orgs/civicpulse/packages, find the geo-api package, link it to the CivicPulse/geo-api repository, and set visibility to public. Then retry.
